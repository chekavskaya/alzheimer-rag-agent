## Alzheimer's Disease Target Discovery RAG-Agent
![alt text](image-1.png)
Прототип RAG-системы, разработанный для помощи исследователям в поиске и анализе потенциальных терапевтических мишеней болезни Альцгеймера на основе актуальных научных публикаций.

## Структура проекта

* **data_collection_eda.ipynb**
  Сбор данных (PubMed API), очистка и эксплораторный анализ.

* **rag/**

  * **retrieval.py** — Поисковый движок на базе FAISS и эмбеддингов all-MiniLM-L6-v2.
  * **generator.py** — Модуль генерации ответов через OpenRouter API (GPT-4o-mini).
  * **pipeline.py** — Retrieval + Prompt Engineering.

* **main.py** — Реализация Streamlit-приложения.

* **requirements.txt** — Список зависимостей.

## Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/your-username/alzheimer-rag-agent.git
cd alzheimer-rag-agent
```

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Настройте API-ключ

Создайте файл `.env` или экспортируйте переменную:

```bash
export OPENROUTER_API_KEY='ваш_ключ'
```

### 4. Запустите приложение

```bash
streamlit run main.py
```

## Оценка качества

Для оценки прототипа использовались следующие метрики:

* **Retrieval Relevance**
  Score в FAISS показывает высокую точность нахождения фрагментов, содержащих названия мишеней.

* **Faithfulness**
  Проверка на наличие галлюцинаций. Система инструктирована отвечать строго по контексту.
  При отсутствии данных агент сообщает о нехватке информации.

* **Druggability Insight**
  Качественная оценка способности модели различать типы препаратов.

## Ответы на вопросы из части 4

### На какие модальности данных можно расширить решение?

Помимо текстов научных статей, систему можно расширить на:

* **Knowledge Graphs**
  Связи ген-болезнь-лекарство из баз вроде Open Targets.

* **Omics**
  Результаты РНК-секвенирования/протеомики из Gene Expression Omnibus.

* **Химические данные**
  SMILES-структуры малых молекул и данные по их аффинности из ChEMBL.

* **Клинические данные**
  Протоколы и результаты клинических испытаний с ClinicalTrials.gov.

---

### Как это можно сделать?

* **GraphRAG**
  Вместо простого векторного поиска использовать графовые базы данных (Neo4j).
  Это позволит модели понимать сложные биологические каскады.

* **Multimodal Embeddings**
  Использовать специализированные модели, например, ESM-2 для белков, чтобы объединить текстовые эмбеддинги с векторными представлениями протеинов в одном пространстве.

* **API Connectors**
  Добавить LangChain Tools, которые в реальном времени подтягивают данные из, например, AlphaFold.

---

### Какие модели и почему вы выбрали для решения?

* **Эмбеддинги: all-MiniLM-L6-v2**
  **Почему:** Она достаточно легкая для запуска на CPU, но при этом хорошо справляется с семантическим поиском в биомедицинских текстах.

* **Генератор: gpt-4o-mini через OpenRouter**
  **Почему:** Модель обладает высоким уровнем reasoning, что критично для выделения мишеней из сложного научного текста. Версия mini выбрана из-за высокой скорости работы и низкой стоимости при сохранении точности, достаточной для RAG-задач.

* **Векторный индекс: FAISS**
  **Почему:** Индустриальный стандарт для быстрого поиска по сходству — позволяет мгновенно обрабатывать тысячи чанков данных.
